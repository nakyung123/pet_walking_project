# 배포 (ship)

백엔드와 프론트엔드 테스트를 실행하고, 전체 통과 시 자동으로 커밋 및 푸시한다.

## 워크플로우

1. **백엔드 테스트 실행**
   ```
   cd backend && npm test
   ```

2. **프론트엔드 테스트 실행**
   ```
   cd frontend && flutter test
   ```

3. **테스트 실패 시**: 실패 내용을 보고하고 즉시 중단. commit/push 하지 않는다.

4. **테스트 전체 통과 시**: 아래 순서로 실행한다.
   - `git status`와 `git diff`로 변경 내용 파악
   - `git add -A`로 전체 스테이징
   - 변경 내용을 분석해 한국어로 적절한 커밋 메시지 자동 작성
   - `git commit` 실행
   - `git push origin <현재브랜치>`

5. push 완료 후 결과 요약 출력 (변경 파일 목록, 커밋 메시지, 브랜치명)
